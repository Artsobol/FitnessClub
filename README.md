### FitnessClub - Система управления фитнес-клубом

Данный проект представляет собой серверную часть (Backend) веб-приложения для автоматизации бизнес-процессов фитнес-клуба. Система позволяет управлять пользователями, расписанием тренировок, абонементами и записями клиентов.

---

### Цель проекта
Спроектировать и реализовать надежную архитектуру серверной части, обеспечивающую взаимодействие различных типов пользователей (клиент, тренер, менеджер) с функционалом клуба.

---

### Задачи и статус реализации

#### 1. Проектирование модели данных ✅ (в процессе)
*   Реализована базовая модель пользователя (`User`) с поддержкой ролей.
*   Реализованы сущности для работы с тренерами: `TrainerProfile` и `TrainerSpecialization`.
*   Запланированы сущности: Абонементы (`Subscription`), Тренировки (`Workout`), Записи (`Booking`).

#### 2. Регистрация и авторизация ✅
*   Интегрирован **Spring Security**.
*   Реализована аутентификация на основе **JWT (JSON Web Token)**.
*   Настроено разграничение прав доступа на основе ролей (`RBAC`).

#### 3. Функционал для типов пользователей 🔄 (в разработке)
*   **Клиент:** Возможность просмотра личных данных и выбора ролей.
*   **Тренер:** Управление профилем и специализациями.
*   **Менеджер/Админ:** Инфраструктура для управления пользователями.

#### 4. Валидация, обработка ошибок и логирование ✅
*   Используется **Spring Validation** (Jakarta Bean Validation) для проверки входных данных.
*   Реализована иерархия исключений (`BaseException`, `BadRequestException`) для единообразной обработки ошибок API.
*   Настроено логирование процессов безопасности и доступа к БД.

#### 5. База данных и миграции ✅
*   Используется **PostgreSQL** в качестве основной БД.
*   Интегрирован **Liquibase** для управления миграциями схем данных.
*   Настроены автоматические обновления схем при запуске приложения.

#### 6. Тестирование API 🔄 (запланировано)
*   Подготовка инфраструктуры для Unit-тестов (JUnit 5, Mockito).
*   Реализация интеграционных тестов с использованием Testcontainers.

### Технологический стек
*   **Java 21**
*   **Spring Boot 4.0.2**
*   **Spring Data JPA** (Hibernate)
*   **Spring Security** (JWT)
*   **Liquibase** (Database migrations)
*   **PostgreSQL** (Database)
*   **Lombok** (Code generation)
*   **Maven** (Build tool)

---

### Структура проекта
*   `feature/user` - логика работы с пользователями и ролями.
*   `infrastructure/security` - конфигурация безопасности, JWT фильтры и провайдеры.
*   `exception` - кастомные обработчики ошибок и исключения.
*   `resources/db/changelog` - скрипты миграции Liquibase.

---

### Как запустить
1. Склонируйте репозиторий.
2. Настройте подключение к PostgreSQL в `src/main/resources/application.yaml`.
3. Соберите проект: `./mvnw clean install`.
4. Запустите приложение: `./mvnw spring-boot:run`.
